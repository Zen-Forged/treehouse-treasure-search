What's new in v2 vs v1

  • Postcard mall card primitive — paper-warm card with "from:" italic eyebrow + Lora mall name + pin/address row + leaf postage stamp with cancellation hash marks. Ships to Home, Saved, and Map identically.
  • Wordmark hero masthead — script wordmark renders ~86px tall on tab pages (no back/right-slot to share the row with). Sub-pages still use the existing 40px inline StickyMasthead.
  • BottomNav drops to 4 tabs — Home / Map / Saved / Profile. Booths tab retires entirely because the persistent mall card already serves the "where am I shopping" function and the saved-finds list groups by booth; the Booths-as-list tab becomes redundant.
  • Saved glyph = leaf — same brand glyph as the save bubbles on tiles; replaces FlagGlyph in BottomNav. Count badge in green pill.
  • Search bar — binoculars + "Search finds, booths, or styles" placeholder ("or styles" hints at R16 tag-based search; signals the tag layer landed in 105 is now searchable).
  • No bottom card on Map — the V1 bottom-card spec dissolves because the persistent top card already identifies the mall. The map fills the body. Tapping a different pin re-scopes (top card updates).

How the map works in v2

The mall card up top is the scope identifier and the mall picker (tap → MallSheet, matching today's MallScopeHeader behavior). Two scope states:

  • Specific mall scope (shown in the third phone): card shows that mall, map zooms to its location with a single highlighted pin + surrounding cartographic context.
  • All Kentucky scope (shown in the scope-variants pane below): card shows "All Kentucky locations" with a map-themed stamp, map zooms out to show all pins. Tapping a pin re-scopes (card animates to that mall, map zooms in).

This dissolves the Frame B carousel + Frame C editorial card from v1 — the persistent top card replaces both.

Open scoping question for design record

Booths tab retirement is a real call. Today /shelves serves three jobs:

  • Browse all booths in a mall — covered by Map (tap a pin → scope to that mall → "explore the booth →" CTA per booth on Saved/Map → /shelf/[slug]).
  • Add Booth (admin/vendor) — relocate to /admin Locations tab and EditBoothSheet (already exists per session 74).
  • "My Booth" entry point for vendors — Profile tab can host a "My booth" link inline (already partially the case via /my-shelf).

Recommend retiring Booths tab. If beta feedback surfaces friction, easy to add a 5th tab back. Lock decision in the design record.

Composition

A primitive component <PostcardMallCard> shipped to Home / Saved / Map.

  • Surface — postcard-bg (#f4ecd8, slightly more saturated than paperWarm), 12px radius, hairline border, shadow-postcard token.
  • "from:" — Lora italic 18px, inkMuted. Postal vocabulary anchor.
  • Mall name — Lora 28px / 500, inkPrimary. Up to 2 lines via WebkitLineClamp; ellipsis if longer.
  • Address row — pin glyph (Lucide MapPin, 13px, inkMid) + address (system-sans 13px, inkMid). Single line, ellipsis.
  • Stamp — 38×44px green rectangle, leaf glyph in green-on, perforated edge via radial-gradient pseudo-elements. Decorative; not interactive.
  • Cancel marks — 3 horizontal bars at postmark opacity (0.34) to the left of the stamp. Mimics a real postal cancellation.
  • Bottom hairline — dashed repeating-linear-gradient at the card bottom, mimicking a postcard's printed divider.
  • Tap behavior — the entire card is tappable; opens MallSheet (existing primitive) for scope change.

Data inputs

  • mall.name — already on malls.name.
  • mall.address_line — composes from address + city + state + zip.
  • (No image needed in the card itself — different from V1's bottom-card spec.)

Surfaces

  • Home (app/page.tsx) — replaces today's MallScopeHeader.
  • Saved (app/flagged/page.tsx) — replaces today's MallScopeHeader.
  • Map (new app/map/page.tsx) — same primitive, drives the map's scope.
