Edit Listing — v1.2 (new page)

Brand-new page. Vendor taps their own find, sees an Edit icon in the corner, lands here. Looks almost identical to Preview, but every field is pre-filled with what they already posted. Changes save as they type.

What you're deciding

  • The layout is deliberately the same as Preview. A vendor editing a find should feel like they're looking at the same page that created it. Same photo, same fields, same order. Only three things are different: a "Replace photo" button on the photo, a new "Status" field, and a quiet "Remove from shelf" link at the very bottom.
  • Autosave. No Save button. Type a price, wait about a second, it saves. A small "Saved" checkmark appears next to the field label when it does. Frame 2 shows this happening.
  • Photo replacement is the one exception. Swapping a photo is a big change, so it gets its own confirmation step (not shown in these frames — triggered by tapping the "Replace photo" button on the photo itself). Rest of the fields autosave without a confirm step.
  • "Available" / "Sold" toggle. Two pills side-by-side. Active one has a dark ring around it. Tapping it flips immediately, shows a small confirmation below. Frame 3 shows the sold state — photo goes slightly grayscale to match how shoppers will see it on Find Detail.
  • "Remove from shelf" as a quiet text link, not a big red button. Destructive actions shouldn't be easy to hit accidentally on mobile.

How a vendor reaches this page

From their shelf or from the feed, they tap their own find. On the Find Detail page, because it's their find, they see a small pencil "Edit" icon in the top-right (where the Save heart would be for other people). Tap it → lands here. Nobody else sees the Edit icon.

Autosave — what the vendor feels

Type in any field. About one second after you stop typing, the page saves in the background and the little green "Saved" checkmark appears next to that field label for a couple seconds, then fades. If saving fails (bad connection etc.), an amber note appears above the field saying "Couldn't save — retry?" — same warning style you already have for the AI caption failure on Add.

"Sold" vs "Found a home"

On THIS page (edit) the toggle says "Sold" — shorter, clearer, inventory-management voice. On the PUBLIC Find Detail page, when a shopper views a sold find, the app shows "Found a home" — the brand voice. Same database record, different audiences. The small confirmation line in Frame 3 makes this connection visible so the vendor knows what shoppers will see.

Three quick calls for you after you see it on your phone

  1. The "Replace photo" button on the photo itself. Dark pill, top-left corner of the photo. Good spot, or would you rather it be a text link somewhere else (under the photo, or in the fields section)?
  2. The status confirmation line in Frame 3. "Marked sold. Shoppers will see this as 'Found a home.'" — is that helpful context, or overkill? Alternative is to just flip the toggle silently and skip the confirmation.
  3. "Remove from shelf" at the bottom. Red italic text link. Is that quiet enough, or too easy to miss? Alternative: same text link but after tapping it, a confirmation sheet slides up asking "Remove this find permanently? This can't be undone." before actually deleting.
