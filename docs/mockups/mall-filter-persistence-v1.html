Existing MallSheet (the one picker affordance — opened from any of the three headers)

Tapping the scope name + chevron on any tab opens the existing <MallSheet> bottom sheet (shipped session 20). "All malls" at top is the clear-filter affordance — no separate clear-glyph needed (D7).

One sheet, three callsites. The same <MallSheet> with the same onSelect(mallId | null) contract serves all three pages. Each page reads treehouse_saved_mall_id on mount, opens MallSheet on header-tap, writes the new value, calls onClose(), and re-filters its own data.

"All malls" = clear filter (italic row at top, already exists). No second clear-affordance. No glyph badge. Active row is text-link-with-hairline (matches existing pattern).

Per-tab counts shown in the right column adapt to the tab: Home shows post counts (existing), Booths shows booth counts, Find Map shows saved-find counts. Same MallSheet props, different findCounts map per consumer.

Find Map empty state when filter excludes all saves

Implementation footprint (what changes when you sign off)

  • New primitive: components/MallScopeHeader.tsx — eyebrow + tappable bold name + chevron + optional geo/count line. Three callsites: Home (replaces the inline FeedHero block), Booths, Find Map. Props: { eyebrowAll, eyebrowOne, mallName, geoLine, onTap } — consumer decides eyebrow strings ("Finds from", "Booths at", "Saves at") so the primitive stays content-agnostic.
  • New hook: lib/useSavedMallId.ts — wraps safeStorage read/write of treehouse_saved_mall_id. Returns [savedMallId, setSavedMallId]. Each tab's page calls it on mount; no cross-tab event subscription (D9).
  • app/page.tsx (Home): swap the existing inline FeedHero scope block to the new <MallScopeHeader>. Behavior unchanged — same persistence key, same MallSheet. Refactor only.
  • app/shelves/page.tsx (Booths): add <MallScopeHeader> below masthead. Read savedMallId on mount. When set, filter the vendor query to that mall and drop the per-mall <mall-section> headers (flat grid). When null, current grouped-by-mall behavior is preserved. effectiveFilter from the Bookmarked-chip work composes — if both filter chip = "Bookmarked" AND savedMallId set, intersect the two.
  • app/flagged/page.tsx (Find Map): add <MallScopeHeader> below masthead. Filter posts query by mall_id when savedMallId set. Empty-state markup when filteredPosts.length === 0 && allPosts.length > 0: "No saved finds at [MallName]. Show all malls to see them." The "Show all malls" link calls setSavedMallId(null).
  • D10 Booths self-vendor gate: when savedMallId set AND viewer is a vendor whose own booth is in a different mall, that vendor's own tile is omitted from the grid (filter applies uniformly). My Booth nav tab is unaffected — it remains the personal escape hatch.
  • No DB / RLS / API / migration changes. Same key, same MallSheet, same data model. Pure presentational + persistence-shape work.
