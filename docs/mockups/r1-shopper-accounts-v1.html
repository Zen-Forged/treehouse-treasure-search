Q1 — What is the profile destination when authed?

Three postures, all consistent with B+E+G. Each shows the AUTHED state as the most defining surface.

  • A — No destination (Pocket-pure). Profile is plumbing. Tap masthead profile bubble → small overlay with email + sign-out + manage. No /me page. Saved tab is unchanged from current.
  • B — Reflective destination (Glass-shape). Profile bubble opens /me. Page shows handle + initials avatar + "scouting since [month]" + private stat row. Saved tab unchanged.
  • C — Saves-as-journal. Profile destination IS the saves grid. Compact identity at top, saves tile grid below. Eats the Saved tab; BottomNav becomes Home + You. Reverses session 109's profile-from-BottomNav demote — call it out before locking.

Q2 — Authed masthead profile bubble: glyph or initials?

  • i — Always CircleUser glyph regardless of auth state. Most invisible; relies on overlay/destination to communicate state.
  • ii — Swap to initials in v1.green when authed (matches frames B + C below). At-a-glance signal that you're signed in.

My instinct: ii with B. Initials make "signed in" legible without a destination check; the green-circle treatment matches the brand vocabulary.
