R10 — Postcard mall card · v3

Component-isolated. Full-height stamp on the right, simpler rendering, and the stamp glyph changes per tab to mirror the BottomNav's active icon — so the card communicates both where you're shopping and what you're doing in one surface.

v3 mockup · 2026-05-04 · just the card · 4 tab variants · stress-tested

What changed from v2

  • Stamp moved to the right side, full content-height — was top-right corner at 38×44, now 56px wide and stretches the card's content height (~120–140px). Reads more like a real postage stamp on a real postcard.
  • Perforated edges retired — v2 used a radial-gradient pseudo-element trick that renders unevenly across browsers. v3 uses a simple 1.5px dashed border on the outer paper-warm rim. Renders identically on Chrome, Safari, Firefox; identical on iOS PWA + Android Chrome.
  • Stamp glyph is contextual — Home tab → house glyph; Map tab → pin glyph; Profile tab → user glyph; Saved tab → leaf glyph. Same Lucide shapes / 2.0 stroke as the BottomNav, scaled to 26px.
  • Cancel marks survive — simplified to 3 plain hairline divs at ~0.65 opacity, slight diagonal rotation. No SVG, no clip-path, no pseudo-elements. Reliable everywhere.
  • Mall name dialed from 28px to 26px — frees vertical room for the stamp to feel proportional. Still the dominant typographic element on the card.

The two-axis read

The card itself = where you're shopping (mall name + address). The stamp glyph = what you're doing (which tab). At a glance the user gets both signals from one surface — the card stays a constant landmark while the stamp confirms the section.

Reinforces the BottomNav: stamp glyph and active tab glyph match. The card isn't a fifth chrome element competing with the masthead and BottomNav — it's part of the same vocabulary.

Surface

  • Card bg — postcardBg #f4ecd8 (new token; warmer than paperWarm).
  • Border — 1px solid rgba(42,26,10,0.10).
  • Radius — 12px.
  • Shadow — shadowPostcard (new token: 0 4px 14px rgba(42,26,10,0.16), 0 1px 2px rgba(42,26,10,0.08)).
  • Padding — 16px 16px.
  • Layout — flex row, gap: 14px, align-items: stretch.

Text block (left)

  • "from:" — Lora italic 18px / inkMuted / line-height 1.
  • Mall name — Lora 26 / 500 / inkPrimary / line-height 1.15. Up to 2 lines via WebkitLineClamp.
  • Address row — pin glyph (Lucide MapPin 13px / inkMid) + system-sans 13px / inkMid. Single line, ellipsis.

Cancellation marks (middle)

  • 3 plain divs, height: 1.5px at postmark color (rgba(42,26,10,0.34)) with 0.65 opacity.
  • Widths 32 / 26 / 30 — slightly varied for hand-stamped feel.
  • Container rotated -4deg for diagonal "smacked" cancellation read.
  • Center-aligned vertically with the text block.

Stamp (right)

  • Outer — paperWarm rim, 1.5px dashed greenBorder for the perforation read, 4px radius, 4px padding. No pseudo-elements.
  • Inner — green fill, 2px radius, glyph centered.
  • Glyph — 26px, greenOn stroke, 2px stroke-width. Source: same Lucide shape as the active BottomNav tab; for Saved, the brand leaf SVG.
  • Width — fixed 56px. Height = card content height via align-self: stretch on the flex parent.

API

<PostcardMallCard mall={mall} stampGlyph="home" | "map" | "profile" | "saved" />

Each consuming page passes its own stampGlyph matching its tab key. The card itself is decoupled from routing — it's a presentation primitive. Tap behavior (open MallSheet) wires up at the consuming-page level.
