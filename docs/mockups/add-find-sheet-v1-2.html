Add Find Sheet — v1.2 exploration

Retire /post as a standalone page. Add a find starts as a bottom sheet from /my-shelf, hands off to /post/preview once a photo is picked.

What this mockup is asking you to judge

  • Does the two-option sheet feel honest or thin? The <MallSheet> is a list of 29 malls; this is two rows. Look at it on your phone — does it earn the "sheet" gesture, or does it feel like a modal that wanted to be a pill-pair?
  • Is the subhead "Photograph something worth keeping." right? It's reaching for the tagline anchor without narrating it. Alternatives noted below.
  • Do the row labels + italic subtitles feel redundant? Row 1 says "Take a photo" + "Use your camera." Row 2 says "Choose from library" + "Pick from your photos." The two-line treatment matches <MallSheet> rows; the one-line stripped-down alternative is noted in Frame 3.
  • Is the glyph weight right? Camera + ImagePlus at 22px, ink-primary. If these read as heavier than the MallSheet pin glyphs, we rebalance.
  • Retire path confirmed? /post becomes a thin redirect to /my-shelf?openAdd=1, preserving inbound links. Admin posting uses /my-shelf?vendor=id. Posting tips retire entirely.

Pattern lineage

The sheet reuses <MallSheet>'s shape verbatim: 20px top radius, paperCream bg, drag handle, IM Fell 22px title, IM Fell italic 14px subhead, hairline divider, rows with glyph + stacked label/sublabel. Framer y-animation uses left:0 right:0 margin:0 auto centering (not transform), per DECISION_GATE Tech Rule on combining centering transforms with y animations.

What disappears

/post's four-stage state machine, the duplicate title/caption/price form (lives on /post/preview now), the vendor-profile setup branch (can't fire — /my-shelf is auth-gated), the Posting tips block, and the Georgia + legacy #f0ede6 palette. /post survives as a 10-line redirect shim so old links don't 404.

Handoff

Either row selected → opens OS camera/picker → vendor picks image → image cached in postStore → sheet dismisses → router pushes /post/preview → AI generation runs inside the preview page's existing loading state. Retires /post's "Reading your find…" state entirely — one loading screen, not two.

Three calls to make after you see this on your iPhone

  1. Proposal vs. stripped. Frame 2 (with italic subtitles) or Frame 3 (labels only)? MallSheet uses subtitles (city/state on mall rows); removing them here would break pattern parity but might feel right since each row here is an action, not a place. I lean Frame 2 for consistency — but this is a live call.
  2. Subhead copy. "Photograph something worth keeping." reaches for the Treasure-the-Find fragment without saying it. Alternatives to consider: "A new find in two taps." (functional), "Show the world your find." (action), or no subhead at all (Frame 3). The design-system rule is: Mode C names the moments that matter, and a subhead is one of those moments — so I prefer it exist, but the copy is open.
  3. Retire /post entirely, or keep shim? Shim is ~10 lines and keeps any inbound /post link working. I recommend shim. The full delete can land in a post-beta cleanup sweep once inbound references are audited.
