Three structural frames for the destination page

Each frame asks the same question: "how should the taxonomy be presented?" They are not style variants — they are three different mental models of what discovery is. Frame A treats discovery as a flat library. Frame B treats it as a hierarchical filing cabinet. Frame C treats it as a guided editorial gallery with library access below.

Read-and-pick

  • Frame A — Flat tag wall. Cheapest to ship, most "treasure-hunt feeling," but loses hierarchy. Counts are visible only by scanning. Fits Treehouse if discovery is meant to feel like flipping through a card catalog.
  • Frame B — Hierarchical rows. Most "professional," easiest to extend with new axes (e.g. by mall, by booth, by price-range). Horizontal scroll consumes vertical space. Fits Treehouse if shoppers think in axes (material/era/object) the way they think in malls.
  • Frame C — Editorial + chip rail. Most "Treehouse-voice" — collections are an editorial moment, chips are a fallback. Highest ongoing maintenance (curated collections need a human or admin tool). Fits Treehouse if discovery is treated as storytelling first, taxonomy second.

What each implies for the data + AI layer

  • Frame A: needs raw chip counts only. posts.tags text[] column populated by enriched /api/post-caption. Cheapest. ~+150 tokens per existing vision call.
  • Frame B: needs typed attributes, not a flat tag list. posts.material, posts.era, posts.object_type, posts.color. Requires committing to a schema before enrichment can be useful. Slightly more upfront design.
  • Frame C: needs Frame A's data plus a collections table (admin-curated picks: title + post_ids + cover thumbs + cadence). Adds an admin tool, but unlocks editorial voice.

What each implies for pagination

  • All three: tapping a chip routes to /?tag=brass (or ?material=brass for Frame B) — a narrowed Home feed. Pagination kicks in only inside that narrowed scope, not on the chip page itself. Existing infinite-scroll pattern can stay; it just runs against a smaller set.
  • The chip page is small (one screen, no pagination of its own). Pagination is a consequence of narrowing, not a parallel concern.

Recommendation

  • Frame C with Frame A's data layer. Ship the chip rail backed by a flat tags text[] column (cheap enrichment), and stub the editorial cards as a single hand-curated collection at first ("This week's brass"). Defer the collections admin tool until the editorial cadence is proven manually.
  • If the editorial layer doesn't earn its keep after 3–4 weeks of beta, retire it back to Frame A. The chip rail is the durable part.
  • Frame B becomes interesting only if shoppers report the flat tag wall feels noisy — at which point we promote the underlying tags text[] values into typed columns. Defer until evidence demands it.
