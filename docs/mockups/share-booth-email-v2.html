Window email — v2.2 masthead + copy refinements

Building on Variant B (post-it embedded in banner). Three refinements landed: (1) masthead shrunk so the booth leads, (2) sender-voice line replaced with universal copy that works without a first name, (3) eyebrow ties email voice to the /shelf app voice.

What changed in v2.2 (all per David's session-51 design review)

1. Masthead shrunk

  • Fix The Georgia 26px/600 "Treehouse Finds" lockup was competing with the booth-name hero. It's supposed to be a quiet brand anchor, not a page title. Three mocks below compare SMALL (13px uppercase wordmark), MEDIUM (17px semibold), and OLD (26px — current production, for reference). David's recommendation favors shrinking.

2. Sender voice line replaced — and a bug closed

  • Bug The current email renders "{senderFirstName} sent you a Window into {vendor}." The server route passes vendor.display_name as senderFirstName, not the authed user's first name — so David's test sends read "Kentucky Treehouse sent you a Window into Kentucky Treehouse." Duplicated vendor name. It was never actually wired to pull a first name.
  • Copy New opener line: "You've received a personal invite." Always works. No sender name to resolve means no bug surface, and identical copy runs for admin sends, vendor sends, and shopper (anonymous) sends. Eliminates the senderMode: "vendor" | "anonymous" branching in the email template (session-50 Q-008 work) — mode is still tracked server-side for rate-limit buckets but the email template stops caring who sent it.
  • Copy Eyebrow above vendor name: "Step inside a curated booth from" — echoes /shelf's BoothTitleBlock eyebrow ("a curated shelf from"). Ties the email voice to the app voice, so a recipient who opens the email and then lands on /shelf hears the same narrator in both places.

3. Implications + what this simplifies

  • Server route stays mostly unchanged. senderMode still gates rate-limit buckets (5/10min auth vs 2/10min anon) — that logic is load-bearing. Only the email template stops reading the mode.
  • ShareBoothWindowPayload simplifies. senderFirstName and senderMode become optional / unused for display purposes. Safe to mark deprecated and delete in a follow-up cleanup.
  • Plain-text fallback adopts the same opener. "You've received a personal invite. \n A curated booth from {vendorName}. \n ..."
  • Preheader simplifies. Current: "{sender} sent you a Window into {vendor}" (anon-branch: "A Window into {vendor}"). New: "A personal invite to a curated booth." One line, always true.

What stays the same as v2

  • Fix Variant B banner — post-it embedded at 86×86 rotated 6°, bottom-right inside the banner. No overhang.
  • Fix Real PinGlyph SVG replaces ⦲.
  • Fix IM Fell 32px vendor name (via Google Fonts <link>) instead of Georgia 34px semibold. Matches /shelf exactly. Outlook / some Android clients fall back to Georgia — graceful degradation.

Three questions for your call

  • Q1 Masthead size: SMALL / MEDIUM / keep OLD?
  • Q2 Masthead style when SMALL: Uppercase wordmark ("TREEHOUSE FINDS" 13px 600 letterspaced), or lowercase/title-case italic variant? Shown below is uppercase; reply if you want a lowercase italic mock.
  • Q3 Tagline under masthead — keep or retire? The "Embrace the Search. Treasure the Find." subtagline adds two lines of vertical space. It's also the email subtagline that Q-005 shortened from three clauses to two. Options: (a) keep it, (b) retire it entirely when masthead shrinks (reduce masthead to one line), (c) keep only when masthead is MEDIUM or OLD, drop when SMALL.
