Frame C — what this looks like in use:
  • Open /review-board → scroll through Foundations to set context → continue down to tile categories → write feedback referencing the tokens you just scrolled past.
  • Anchor pill lets you jump back to Foundations without scroll. Sticky during scroll.
  • One long page; no nav state to manage. Reference is "above" not "beside," so cross-referencing requires scroll back up while writing feedback.

Trade-offs at a glance

  • Frame A (top tabs) — cleanest separation, familiar pattern, tab-bar grows naturally. Only sees one context at a time.
  • Frame B (sidebar) — persistent reference, fastest section-switching, shows counts. Eats horizontal space; weaker on narrow viewports.
  • Frame C (anchored above) — simplest, no nav state, preserves existing tile layout untouched. Long scroll; reference is "above," not "beside" tiles during feedback writing.

Recommendation

  • If the primary use-case is review a tile, then check the standard, Frame A is the right shape (sequential).
  • If the primary use-case is have the standard visible while reviewing, Frame B is the right shape (simultaneous).
  • Frame C is the cheapest to ship but pays an ergonomic tax every time you write feedback.

Pick A, B, or C and I'll surface Q2 (sections) + Q3 (token vocabulary mechanism) as prose. Then design record lands + arc 2 ships next session.
