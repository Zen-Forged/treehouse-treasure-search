Booth management — admin

Surfaced from David's Gemba walk at America's Antique Mall. Items #2 + #3 + delete + (post-mockup addition) admin-add affordance. Pencil bubble repurposes from a misleading link-to-vendor-profile into the canonical edit affordance, opening a bottom sheet that mirrors DeleteBoothSheet's pattern on the same page. A new "Add a booth" tile at the end of the grid completes the management trio (Add · Edit · Delete) on a single demo-friendly surface — admins no longer need to leave /shelves for booth setup. Historical note: AddBoothInline lived on /shelves in early sessions, was removed twice (sessions 37 + 67) on the reasoning that admin work belongs on /admin. This third pass is intentional — David's demo posture wants the pattern back, and the Admin page is meaningfully "backend" not vendor-presentable.

Decisions D1 – D7

#DecisionRecommendation
D1 Repurpose Pencil to open the edit sheet, vs add a separate edit button next to it? Repurpose Pencil. Pencil = edit is the universal mental model. Today it links to /vendor/[slug] — read-only public profile, weak match for the icon. Profile remains reachable via direct URL or from /shelf/[slug].
D2 When display_name changes, auto-update slug? Auto-update. Bookmarks and finds reference vendor_id, not slug; URL change is the only consequence and is the expected behavior on rename.
D3 Allow edit on claimed booths (user_id != null)? Yes. Editing labels doesn't strand the auth user the way deleting does. Keep DELETE's safety gate; drop it on PATCH.
D4 Allow mall change (David's #3)? Yes. Conflict path: vendors_mall_booth_unique can fire 23505. PATCH catches it and returns a clean message — sheet shows the conflict pill (Frame 3) instead of bubbling a 500.
D5 Include hero photo edit in this sheet? No — out of scope. Admin already has /my-shelf?vendor=<id> as the canonical hero-edit surface. Keeping this sheet tightly scoped (3 fields) means a single PATCH endpoint and shorter Save path.
D6 Where does the new "Add a booth" tile sit? Single tile at the very end of the whole grid (after all mall sections), admin-only. Mirrors how "Add a find" sits at the end of /my-shelf's booth grid (per item #7 we just shipped). Per-mall-section tiles would multiply the affordance for no benefit; floating-action-button breaks the masthead-anchored chrome pattern.
D7 Add sheet pattern — bottom sheet (matches Edit) or inline expansion (existing AddBoothInline)? Bottom sheet. Add + Edit + Delete on /shelves share the same sheet ergonomics — coherent admin-management trio. Inline expansion would create two visual paradigms on one page. Add sheet keeps AddBoothInline's field shape (mall · booth_number · display_name · optional hero photo, kept here unlike Edit per D5 since hero is most easily captured at create-time during the mall walk). Pre-selects the active mall filter (savedMallId) when present.

Flow

Admin on /shelves taps Pencil on a tile. Sheet slides up. Form pre-fills with the booth's current values. Admin edits. Save commits + closes sheet + tile re-renders with updated values.

Implementation

Components + endpoint sketch (post-approval). Mockup is the commitment surface; build spec gets written after Save.

UI — Edit EditBoothSheet component on /shelves. Mirrors DeleteBoothSheet's sheet pattern (backdrop, grabber pill, motion-y bottom-up entry, body-overflow lock). 3 form fields: mall select · booth_number input (autoCapitalize="characters") · display_name input.
UI — Add AddBoothSheet component, same shell as EditBoothSheet. 4 fields: mall select (pre-selected to savedMallId when active) · booth_number · display_name · optional hero photo. New admin-only AddBoothTile rendered after the last mall section. Both sheets share field markup via a small BoothFormFields sub-component to avoid drift.
State Edit: form pre-fills from vendor; Save disabled until any field differs from initial; changed visual flag on diffed inputs. Add: form starts empty (mall pre-selected); Save disabled until display_name + mall present. Both: Cancel discards.
API — Edit New PATCH /api/admin/vendors. Auth gate: requireAdmin. Body: { vendorId, display_name, booth_number, mall_id }. Auto-derives slug from display_name. Catches Postgres 23505 → returns { error, code: "BOOTH_CONFLICT" } with 409. No safety gate on user_id (D3).
API — Add Reuses existing createVendor + /api/vendor-hero pipeline (same code path AddBoothInline already exercises). No new endpoint needed. Conflict shape: 23505 → conflict pill, identical to Edit.
Wire Pencil bubble at app/shelves/page.tsx:169-175 swaps from <Link href="/vendor/[slug]"> to <button onClick={() => setEditTarget(vendor)}>. AddBoothTile rendered after groupByMall output, gated on isAdmin(user). Both sheets + AnimatePresence mounted alongside DeleteBoothSheet at the page root. AddBoothInline retained on /admin Vendors tab unchanged.
Optimistic update Edit: sheet animates out + tile re-renders with new values. Add: sheet animates out + new tile fades into the grid in its mall section. Both via in-place setVendors updates — no full reload.
