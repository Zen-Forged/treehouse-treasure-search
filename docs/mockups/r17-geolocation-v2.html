R17 — Geolocation Enrichment v2

Frame A's twin-button CTAs + Frame B's eyebrow-row pill (small-caps, paper-warm, no glyph).
V2 spans distance copy as the only remaining choice axis. System-wide rollout panel below.

Locked from V1

  • Pill anchor: eyebrow row above the cartographic card, right-aligned (Frame B). Paired with the existing italic Lora "Find this item at" eyebrow on the left.
  • Pill visual: small-caps, paper-warm bg (#ede6d5), 1px ink-hairline border, 999px radius, sys font 10px / 700 / 0.12em letter-spacing, ink-muted color. No arrow glyph.
  • CTA shape: twin buttons below the card — outline "View on Find Map" (with map glyph) + filled green "Navigate" (with arrow glyph). Equal width, 10px gap.
  • Permission-denied state: hide pill + CTAs entirely. Card reverts to current state (silent failure).
  • Pill is decorative (not tappable). CTAs cover both affordances; tappable pill would be redundant with "Navigate."

Q1 — Distance copy (only remaining axis)

  1. α — minimal: "2.7 MI" — terse, requires only haversine math (no API call). Reads as a postmark stamp.
  2. β — verbose: "2.7 MI AWAY" — adds a connecting word; reads more like editorial copy. Same data shape as α.
  3. γ — distance + ETA: "2.7 MI · 8 MIN" — Apple-Maps-style pair. Requires a Mapbox Directions API call per render (cached by mall_id + user-coord-bucket so it's not literally per-render). Strongest "you can be there" signal.
γ trade-off: the ETA is the strongest digital-to-physical signal — "you can be there in 8 minutes" is a concrete decision-trigger. But each render needs a Directions API call. Mapbox bills $0.50 per 1k matrix calls; with caching (5-min TTL on user-coord-bucket × mall_id), expected cost is ~$0.50/month at beta scale. The decision is mostly editorial: does ETA add or subtract from the postcard-stamp feel?
