Rich PostcardMallCard — V3 (Affordance Redesign)

"select location" handwritten label + arrow + dashed-stamp aren't landing in iPhone QA. V3 explores three alternatives for the change-location affordance. Photo + name + address + search bar are unchanged from V2 across all three frames.

What V3 is asking you to judge

Pick a structural shape for the change-location affordance now that the script-label + arrow + dashed-stamp combination is being retired. The card still needs to communicate "this is the mall you're shopping at, tap to switch to a different one" — what's the cleanest way to say that?

Locked from V1 + V2 + V2.1 dials

  • Frame A structure — photo banner above text+affordance row, search bar pill at bottom of card.
  • Photo — 180px tall, 14px inset from card sides + bottom, 8px corner radius.
  • Mall name — Lora 22px (production: fluid 22→16 measure-and-shrink), single row.
  • Address — full street + city + state + zip, tightly grouped under name (margin-bottom 0 dial just shipped).
  • Card-to-grid bottom spacing — 14px breathing room before masonry (just shipped).
  • Empty state — drop photo, keep everything else including the affordance + search.
What's being dropped (David's iPhone QA call):
  • "select location" handwritten Dancing Script label
  • Curved hand-drawn arrow connecting label to stamp
  • Dashed-border green stamp + cancellation ink (Frames α + β); kept in a different role in Frame γ
Net visual effect: the upper-right corner gets simpler. The card stops trying to be a literal postcard with a postal stamp; it stays a paper-stock card with a clear tap target.

Questions for V3 → ship

  1. Frame pick — α (pill "CHANGE LOCATION"), β (pill "4 LOCATIONS" with chevron), or γ (inline chevron on mall name, no upper-right element).
  2. Pill copy (if α or β picked) — "CHANGE LOCATION" / "BROWSE LOCATIONS" / "OTHER LOCATIONS" / "X LOCATIONS"? Easy to refine.
  3. Slim-card consistency — slim <PostcardMallCard> on Map+Saved still uses the italic-Lora "Location" label + dashed-stamp from session 116. If we adopt a pill / chevron on the rich card, should the slim card follow for consistency? Or keep separate vocabularies because the slim card is informational on /map + Saved?
