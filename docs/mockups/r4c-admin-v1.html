R4c — Admin Malls tab

Three-state lifecycle for malls: Draft / Coming soon / Active. New 4th tab on /admin carries the toggle. Shopper pickers filter to active only; admin surfaces show all with status pills; vendor-owned pages still render their own mall regardless.
Session 56 · 2026-04-24 · design record: docs/r4c-mall-active-design.md

Three questions still tentative — flagged in the design doc for your review before implementation

D4 — Migration default. Recommendation: all existing malls default to draft; you flip the 1–2 you care about in the new admin tab post-deploy (≤30 sec). This reveals the pollution problem R4c solves rather than papering over it.

D5 — Where the toggle lives. Recommendation: new Malls tab on /admin, 4th position after Requests / Posts / Banners. Keeps schema-editing admin colocated. /shelves stays a browser, not an editor.

D6 — Can vendors request booths at coming_soon malls? Recommendation: no — vendor-request dropdown filters to active only. Accepting requests before the mall is operational creates a queue you can't act on.
