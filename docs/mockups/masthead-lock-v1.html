Masthead lock

Record 3 of 3 — visual stability with affordances. Wordmark X-position locked across every page; right slot accepts variable affordances within a fixed-width column. Booth-page internal scroll flattens. /post/preview custom masthead unifies with StickyMasthead.

Frozen decisions (D12–D18)

  • D12. Inner grid = 1fr auto 1fr on every Masthead instance. Wordmark sits in the auto middle column, always visually centered regardless of left/right slot content.
  • D13. Wordmark = single render: IM Fell italic 22px on inkPrimary. No per-page variant.
  • D14. Left slot = back button (12px chevron + 28×28 tap target) ONLY on detail/leaf pages (/find/[id], /shelf/[slug], /post/preview, /post/edit/*). Empty on root tabs (Home, Booths, Find Map, My Booth).
  • D15. Right slot = accepts variable content (sign-in icon / sign-out text / Admin pill / bookmark + share bubbles) but locks to min-width: 80px matching the heaviest case. Lighter slots render right-aligned within that width.
  • D16. Booth-page internal overflow-auto scroll containers (/my-shelf, /shelf/[slug]) = flattened to standard document scroll. Masthead behaves identically across all pages.
  • D17. /post/preview custom masthead = replaced with StickyMasthead. Stacked "Review your find" title + italic subhead moves to a content block BELOW the masthead, not inside it.
  • D18. Total height = max(14px, env(safe-area-inset-top, 14px)) + 40px content + 12px bottom. Single locked computation across all pages.

D16 — Booth-page scroll flatten

Today /my-shelf + /shelf/[slug] render the masthead inside an internal overflow-auto scroll container. Sticky behavior is relative to that container, not the document — masthead "stickiness" feels different from every other page. After: container removed; masthead sticky against document scroll like everywhere else.

  • What changes — BoothPage.tsx outer wrapper drops the overflow-auto + height: 100% container. scrollTarget ref + scroll-linked masthead border tracking move to document scroll.
  • What stays — bottom nav (separate from booth page chrome), all booth-page content, BoothHero post-it, tile sections, mall picker.
  • Risk — any code that observed the inner scroll container directly (e.g. for "scroll to top" behavior on bookmark toggle) needs to switch to window scroll. Search-and-fix pass during implementation, not a structural blocker.

What's preserved

Visual stability with affordances (Q2 = locked) means the wordmark + height never move; the right slot can still hold variable content as long as it fits within the locked column.

  • Right-slot content variants supported: empty (Find Map, /post/preview), single icon (Home guest = CircleUser), single text label (Home signed-in = "Sign out"), single pill (Booths = "Admin"), two bubbles (Find Detail = flag + share, Booth Detail = bookmark + share). Heaviest case (~76px) sets the column min-width.
  • Left-slot content variants supported: empty (root tabs), back button (detail/leaf pages). Same logic: empty slot reserves the same column width so wordmark X holds.
  • Per-device variance preserved: env(safe-area-inset-top) still flexes by device (notched vs non-notched). Within a single device, all pages render the masthead at the exact same baseline height.
