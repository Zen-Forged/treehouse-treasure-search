Mall hero in feed — R11 v1

Two compositions for showing a mall's hero photo when the feed is filtered to that mall. All-malls case shows no photo (current behavior).

Decisions already locked

  • Schema: new malls.hero_image_url column via migration 016. Optional. NULL = no hero set for that mall.
  • Storage: reuse site-assets bucket; admin upload via existing pattern (mirrors featured-image route).
  • Admin UI: upload affordance lives inside the expanded /admin Malls tab row (next to the status-toggle).
  • Fallback: when no hero is set OR when filter = "All Kentucky Locations", render the existing MallScopeHeader exactly as today (no photo).
  • Photo container: rounded 8px corners (matches BoothHero v1.bannerRadius), 22px horizontal page padding (matches MallScopeHeader).

What this mockup is asking you to judge

  • Composition: A (photo ABOVE the existing text header) vs B (photo BEHIND the text with white-on-photo overlay).
  • Implicitly: how dramatic should the mall-hero moment be?
