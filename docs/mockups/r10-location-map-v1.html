Question 1 — Which structural direction?

Three frames span coherent design directions, not just style variants:

  • Frame A — Cartographic & curated. Custom map style (warm cream, sage water, hand-drawn feel), leaf pins, single sticky bottom card. Map & list are two views of the same data via toggle. Strongest "this is a Treehouse surface" read.
  • Frame B — Familiar & legible. Standard light Mapbox style, name-pill pins like Airbnb's price-pills, horizontal carousel of mall cards at the bottom. Lowest learning curve for shoppers; map & Booths-list coexist as separate destinations.
  • Frame C — Editorial & sparse. Minimal map base, photo-thumbnail pins, larger single card with editorial framing ("Featured this week from this mall"). Treats the map as an editorial discovery surface, not a directory.

Recommendation: Frame A. Best fit for the "doesn't feel tacked on" requirement — the cartographic style is a canonical extension of the warm-cream/Lora visual language. Pin treatment is a separate axis below; A composes well with leaf, dot, or numeral pins.

Question 2 — Which pin treatment?

Five variants in the pin pane below. Pick one, or pick a state pair (e.g. dot when zoomed-out, leaf when selected):

  • Leaf bubble — leaf glyph in a paper-warm circle outlined in green; selected = green fill. Strongest brand read.
  • Name pill — Airbnb-style pill with mall name + small leaf glyph. Most legible at first glance, but visually heavier; risks Airbnb-mimic.
  • Photo thumbnail — 50px circular hero photo with leaf badge. Most "at-a-glance personality" but expensive at zoom-out.
  • Numeral pill — booth-numeral typography (Times New Roman) showing # of finds at that mall. Composes with the existing booth-numeral system.
  • Cartographic dot — anonymous green dot, name reveals on tap. Cleanest at zoom-out; least information-rich.

Recommendation: Leaf bubble for default; cartographic dot at deep zoom-out (cluster). Brand glyph + scales gracefully.

Question 3 — Where does the Map tab slot in the BottomNav?

Three options in the nav-placement pane below. (You said "we'll be adding the map marker icon to the bottom nav," so all three are tab-based — no overlay.)

  • Slot 1 — Replace Booths. Map tab IS the new "browse locations" surface; the in-page list ↔ map toggle preserves the list affordance. Cleanest BottomNav (still 5 tabs).
  • Slot 2 — Insert as 5th tab. Map gets its own slot between Saved and Booths. Booths list stays. BottomNav becomes 6 tabs for vendors/admin (still 5 for guests).
  • Slot 3 — Replace Profile, push Profile to masthead. Decouples auth chrome from the primary nav (reverses session 90's relocation). Strong structural argument but undoes a recent decision.

Recommendation: Slot 1 (replace Booths). Cleanest mental model: the Map tab IS where you browse locations, and the in-page list/map toggle preserves the list view. Strongest argument against "tacked on" because it doesn't grow the nav.

Cross-domain references pulled (4–6 angles)

Airbnb (your screenshot) — pin-on-map + bottom card + horizontal category rail. Lesson: the genre-defining shape; we mimic the bones, not the price-density.
Hipcamp — warm/outdoor energy, branded pin glyphs (tents), low pin density. Lesson: closest brand cousin; brand glyph beats generic MapKit pin.
Atlas Obscura — illustrated cartographic base, "places to discover" framing, no transaction pressure. Lesson: muted cartographic styling shifts mental model from "buy" to "explore."
AllTrails — pin clustering at zoom-out, individual pins at zoom-in, bottom carousel. Lesson: clustering preserves overview at the state level (KY-wide).
Roadtrippers — photo-thumbnail pins, location-detail bottom card (no "book"). Lesson: Frame C's editorial direction is built on this shape.
Apple Maps Guides — minimal pin styling on muted base, curated POI emphasis. Lesson: zero-chrome map base + strong content cards is a viable direction.

Anti-references: Yelp, DoorDash, Uber Eats. Transactional density wrong for a discovery surface.

Composition

Mirrors the v1.x card vocabulary used on /shelves & /flagged section headers.

  • Photo — 110px square, 8px radius, sourced from malls.hero_image_url (column added in migration 016).
  • Name — Lora 18px / 500 / inkPrimary. Single line, ellipsis if overflow.
  • Address — system-sans 12px / inkMuted. address · city, state ZIP. Single line, ellipsis.
  • Stats — 2 numbers in Times New Roman (booth-numeral typography), Lora-italic labels. Numbers in v1.green.
  • Surface — paperWarm bg, inkHairline border, 14px radius, polaroid shadow. shadow-polaroid token.

Tap card → routes to /mall/[slug] (or new /location/[slug] page if that route gets the v1.x ecosystem rebuild — Frame A's "list view" already provides the nearest equivalent).
