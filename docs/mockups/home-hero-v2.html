Axis

Implementation
One <section>. Pure CSS: position: sticky; top: calc(90px - 33vh). ~1 commit; no JS.
Two siblings: scroll-away hero (~188px) + sticky strip (~90px) with position: sticky; top: 0. Strip has solid cream bg + soft drop shadow. ~1-2 commits; minor markup split.
Single hero element. Scroll listener (or scroll-timeline CSS) drives height + wordmark opacity. ~3-4 commits; scroll-driven animation discipline.
Image continuity
+ Native continuity — same image scrolls into its sticky position; bottom 90px naturally pins. No bg-seam.
− Solid cream strip is visually decoupled from the hero image. Could swap to bottom-cropped image bg for visual continuity but background-position math gets fiddly across viewport widths.
+/− Cover-sized on shrunken 90px container shows the horizontal middle stripe of the image (NOT the bottom). To match Shape A's appearance, would need background-position: bottom dialed in during the collapse.
Wordmark fade
Natural — wordmark scrolls out of viewport with the top of the hero. No opacity animation needed.
Natural — hero scrolls away in normal flow; wordmark goes with it. Strip never had a wordmark.
Animated — wordmark opacity 1→0 over scroll 0→188px. Adds polish but introduces an animation curve to tune.
iOS Safari risk
Low. position: sticky with negative top is supported since iOS 11. iPhone QA validates.
Very low. Plain position: sticky; top: 0 is universally supported.
Medium. CSS scroll-timeline only on iOS 17.4+; JS fallback adds scroll listener + RAF discipline. Risk: jank on slow iPhone PWA scroll.
Search position
Search bar lives at hero's bottom: 32 (per your dial). In scrolled state, search ends up at strip-y ~12-54 (top-biased in the 90px strip). 36px breath below.
Search bar lives in the sticky strip independently. Always centered (top: 22, height 46, breath 22 below) regardless of scroll state.
Search bar's bottom offset animates 32→14 during scroll so it stays vertically anchored inside the shrinking hero.
Hero asset constraint
Hero asset's bottom 30% should be visually quiet so the search bar reads in the sticky state. Your attached asset's bottom is wood-tone — qualifies. Already met.
No constraint — sticky strip is solid cream. Hero asset only matters for the initial-state composition.
Asset's middle horizontal stripe (which is what cover-sizing on shrunken container shows) needs to be visually quiet. Your asset's middle has the wordmark — would compete with the search bar in collapsed state UNLESS background-position: bottom dial fixes this. Worth a sub-mock to validate.
Recommendation
Recommend Shape A. Simplest impl + natural image continuity + matches your description ("background sticks with search bar so it's still a header but thinner not the full hero"). Single Arc dial commit on existing HomeHero.
Fallback if Shape A's bottom-of-image strip reads off-brand or surfaces iOS Safari issues. Solid cream is reliably consistent.
Defer. Most elegant but cost is 3-4× higher and the value-add (smooth animation) is dial-cycle territory not V1 ship territory. Could promote to Tier B7 in design record if Shape A ships clean but you want smoother transition later.
