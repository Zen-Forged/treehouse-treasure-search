Production

Commission or AI-generate (Midjourney + post-process) 1 hero photograph: warm-cream textured paper background + flatlay scene with iPhone, botanicals, antique postcard, brass key, treehouse sketch.

Optimize the hero to ~400-800KB WebP/AVIF. Phone screen rectangle in the photo defines geometry — production CSS positions a transparent live-app `<div>` exactly over that rectangle.

Engineering

Single PNG/JPG bg via CSS `background-image`. Live PWA's existing 430px max-width column renders as absolute-positioned child at exact pixel coords matching photo's phone-screen rectangle.

Viewport scaling: photo + position scale together via CSS `background-size: cover` + percentage positioning. Phone-screen-rectangle math locks at one breakpoint; smaller viewports fall back to current cream-bg behavior.

Trade-offs

+ Highest brand fidelity. Matches reference image's editorial photography voice exactly. Investor-update / press-page energy.

− Asset production is the load-bearing cost. AI generation works but requires careful prompt iteration + post-processing for brand-true botanicals/postcard.

− Geometry locked to photo. Future "move the phone" or "add 4th decoration" requires re-shooting/re-rendering.

Production

Zero photography needed. SVG phone bezel as React component (inline path). Decorative botanicals/dots/leaves as SVG icons or CSS-rendered geometry. Brand voice expressed via fonts + colors + glyph repetition.

Asset weight: minimal (~10-20KB total for all SVG primitives).

Engineering

Pure React + CSS. `<DesktopFrame>` component composes `<PhoneBezel>` SVG + `<BrandChrome>` + decorative SVG elements absolute-positioned. Live PWA renders inside SVG screen rectangle.

Viewport scaling: SVG primitives scale cleanly via `viewBox` + `preserveAspectRatio`. Independent of any photo geometry. Easy to dial individual elements.

Trade-offs

+ Lowest production cost. Ships entirely in code; no photography commission or AI generation.

+ Future-flexible. Decorations are SVG components; easy to swap, dial, reposition.

− Doesn't match reference image's voice. Reads as cleaner / more designer-tool aesthetic (Linear, Stripe, Vercel). Loses Treehouse's editorial-warmth / antique-postcard brand vocabulary.

Production

1 background photograph (paper-texture/flatlay without the phone in frame). Photo carries brand voice. Phone bezel rendered as React/SVG component on top.

Asset weight: paper-bg ~200-400KB (smaller than Frame A since no phone in shot) + SVG phone ~10KB.

Engineering

Paper bg as CSS `background-image`. SVG `<PhoneBezel>` component composes independently — positioned via flexbox/grid, not tied to photo coordinates. Decorations live in photo (postcards, botanicals).

Phone position dials freely. Future "add 4th decoration" = re-shoot bg only. Future "wider phone bezel" = code change.

Trade-offs

+ Mid production cost. Single bg photograph (without phone) is simpler to source/generate than Frame A's full composition.

+ Phone-position flexibility. Decoupled from photo geometry.

− Phone bezel may read as "computer-rendered against real photo" — slightly less polished than Frame A's photographed phone in scene.

+ Still carries editorial brand voice via photographed bg (vs Frame B's pure-code aesthetic).
