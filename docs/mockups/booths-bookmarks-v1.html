Implementation footprint (what changes when you sign off)

  • Files removed/gated: components/AddBoothInline.tsx stays in tree but its <AdminOnly> wrapper on app/shelves/page.tsx:402 becomes the only consumer; we leave the file as a future admin tool. The inline form is removed from the public Booths page.
  • BottomNav gate change: components/BottomNav.tsx:79–83 conditional flips — boothsTab now appears for all states (guest, vendor, admin). Tab order becomes Home · Booths · Find Map · [My Booth].
  • New primitive: components/BookmarkBoothBubble.tsx — wraps Lucide Bookmark with frosted bubble + saved/unsaved fill state, mirroring the FlagGlyph callsite shape.
  • New helpers in lib/utils.ts: boothBookmarkKey(vendorId), loadBookmarkedBoothIds(), loadBookmarkedBoothCount() — parallel to existing flagKey + loadFollowedIds. Prefix: treehouse_booth_bookmark_.
  • Page touches: app/shelves/page.tsx (add filter chips + bookmark bubble on each card; remove AddBoothInline; gate "Manage" + delete + edit affordances behind existing AdminOnly). app/shelf/[slug]/page.tsx (bookmark bubble in masthead, hidden when own-booth). components/BottomNav.tsx (gate flip — boothsTab now renders for all states; tab order becomes Home · Booths · Find Map · [My Booth]). app/my-shelf/page.tsx — untouched (D12 — vendors use Booths tab like everyone else).
  • No DB / RLS / API changes. All persistence is localStorage; no Supabase migrations, no new endpoints.
