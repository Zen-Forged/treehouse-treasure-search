<PolaroidTile> primitive — v1

Phase 2 Session B · design-system hardening pass · extracts the polaroid wrapper inlined across 6 callsites into a single primitive. Goal: zero visual delta. Each callsite below renders today inside its own inline polaroid block; after extraction they render through <PolaroidTile>.

Decisions to lock

D1 · Wrapper variant API

Two padding patterns observed across the 6 surfaces:

  • 7px 7px 8px — Home, /post/preview, /post/tag (caption integrated inside the polaroid)
  • 7px 7px 0 — /flagged, /shelf/[slug] WindowTile, /shelf/[slug] ShelfTile (caption block outside, in a 76px metadata strip below the frame)

Options:

  • (a) bottomMat: "inside" | "outside" — semantic enum naming the conceptual variant
  • (b) captionInside: boolean — boolean prop, simpler
  • (c) padding: string — pass-through, maximally flexible but loses semantic meaning

My read: (a). The two variants are conceptually distinct ("polaroid with a caption mat" vs. "polaroid as a pure photo, caption sits below"). Enum reads better in callsites than a boolean.

D2 · Photo inner background

Three values observed:

  • v1.postit (#fbf3df) — Home, /flagged, /shelf/[slug] (real-photo placeholder + image-load shimmer)
  • v1.paperCream (#e8ddc7) — /post/preview (review-state contrast against any image bg)
  • #cdb88e warm tan — /post/tag (warm placeholder while photo loads on capture)

Options:

  • (a) photoBg?: "postit" | "paperCream" | "warmTan" — enum, tight
  • (b) photoBg?: string — pass-through any color
  • (c) Tokenize the warm-tan as v1.warmTan + accept any v1 color name

My read: (b). Three values isn't enough to justify an enum; the prop name signals intent. Default v1.postit.

D3 · Tap interaction (Home only)

Home masonry tile has a tap-flash + scale animation on touch (scale(1.028) on tap, returns over 320ms). No other surface uses it.

Options:

  • (a) tap?: boolean — primitive owns the tap state internally
  • (b) Parent owns tap state, passes tapped?: boolean
  • (c) Skip — leave Home's tap behavior outside the primitive (Home wraps the primitive in its own touch handlers)

My read: (a). Self-contained interaction is cleaner; Home can opt in with one prop. If a future surface wants tap, no extra wiring.

D4 · "Last viewed" highlighted state (Home only)

Home masonry highlights the last-tapped tile when the user navigates back, with a 1.5px green border + halo shadow that fades after 1.6s.

Options:

  • (a) highlighted?: boolean — parent controls the state via timeout
  • (b) lastViewedKey?: string — primitive reads from sessionStorage
  • (c) Skip — Home keeps inline rendering of the highlight effect

My read: (a). State management belongs to the parent; primitive just renders the visual.

D5 · Lens filter

Treehouse Lens filter (contrast(1.08) saturate(1.05) sepia(0.05)) is applied on every surface EXCEPT /post/tag. Capture flow shows the raw photo; everywhere else applies the lens.

Options:

  • (a) lens?: boolean default true — opt-out for capture
  • (b) lens?: boolean default false — opt-in for browse surfaces

My read: (a). 5 of 6 surfaces apply the lens; default-on minimizes prop noise on the common case.

D6 · Render slots

The primitive needs two slot positions:

  • topRight — bookmark / unsave / heart bubble (Home + /flagged today). Positioned absolutely top:8 right:8 inside the photo slot.
  • below — content rendered after the polaroid wrapper closes (timestamp on Home; 76px caption block on /flagged + /shelf; italic label on /post/tag). The "outside-mat" variant in D1 is essentially "this slot is non-empty + sized."

API: both pass through as ReactNode children-style props. topRight renders inside the photo slot z-index 3; below renders as a sibling after the polaroid wrapper.

My read: confirm slot shape — any concerns?

D7 · Sizing + carousel variants

/shelf/[slug] ShelfTile uses width: 52vw, max-width: 210, scrollSnapAlign: start for horizontal-scroll carousel. Other surfaces are full-width within their grid cell.

Options:

  • (a) Primitive accepts no sizing props — caller wraps in their own container with the carousel sizing
  • (b) Primitive accepts maxWidth?: number + flexShrink?: 0 + scrollSnap?: boolean

My read: (a). Sizing is layout concern, not chrome concern. Keeps the primitive's API tight; ShelfView wraps in the carousel container as today.

Frame 3 — Proposed API + callsite examples

// components/PolaroidTile.tsx
interface PolaroidTileProps {
  src:     string;
  alt:     string;

  // D1 — wrapper variant
  bottomMat?:  "inside" | "outside";  // default "inside"

  // D2 — photo inner background
  photoBg?:    string;                // default v1.postit

  // D3-D4 — interaction states
  tap?:        boolean;               // default false; Home only
  highlighted?: boolean;              // default false; Home only

  // D5 — lens filter
  lens?:       boolean;               // default true

  // photo render
  aspectRatio?: string;               // default "4/5"
  objectFit?:  "cover" | "contain";   // default "cover"

  // global state
  dim?:        boolean;               // default false; sold

  // photo inner chrome
  innerBorder?:     boolean;          // default false; Home only
  innerInsetShadow?: boolean;         // default false; /post/tag only

  // D6 — render slots
  topRight?:   ReactNode;             // bubble overlay inside photo slot
  below?:      ReactNode;             // sibling after polaroid wrapper

  // image error fallback
  fallback?:   ReactNode;             // rendered inside photo slot on error
}

Token usage in primitive

Reads v1.shadow.polaroid + v1.paperWarm from the Phase 2 Session A token additions. Default photoBg is v1.postit.

Out of scope for this primitive

  • Sizing / carousel container — caller wraps (D7).
  • Image preview cache (sessionStorage treehouse_find_preview:) — stays at the callsite.
  • Link / onClick wrapping — caller composes; the primitive renders just the visual.
  • framer-motion layoutId for shared-element transitions — Track D is currently dormant; if it's revived in a future session, the primitive can grow a layoutId prop then.
