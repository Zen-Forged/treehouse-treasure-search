What I'm asking you to judge

Phase 1 audit found 4 inline empty states drifting on every visual axis except font family. Frame 1 below shows what's shipping today. Frame 2 shows the same 4 surfaces re-rendered with a single proposed primitive. The primitive enforces structure (clearance, sizing, centering); it does not change copy or voice.

Decisions D1–D6 to freeze

D#QuestionMy read
D1 Title font size — currently 20 (Home + /shelves), 24 (/flagged), none (/shelf/[slug]). 22 — splits the difference; Lora at 22 reads as journal, not display chrome.
D2 Title required or optional? /shelf/[slug] ships subtitle-only today. Optional. Either-or-both. Preserves the scenic single-line on /shelf/[slug].
D3 Voice unification — should I rewrite copy across the 4 surfaces? No. Preserve per-surface voice. The voices ARE deliberately different. Primitive enforces structure, not copy.
D4 Subtitle line-height — currently 1.65 / 1.7 / 1.75. 1.7 — median. Balances air without feeling spacy at 14px italic Lora.
D5 Subtitle maxWidth — currently 230 / 280 / none. 260. Keeps single-clause subtitles from wrapping awkwardly; reads as narrative not paragraph.
D6 Clearance — paddingTop currently 72 / 60 / 80 (top-level) / 48 (mid-page nested in /shelf/[slug]). clearance prop with two modes: "masthead" default (= MASTHEAD_HEIGHT + 32 = 135), or a number for nested mid-page contexts. /shelf/[slug] passes 48.

Already settled (no decision needed)

  • Family: FONT_LORA · Title color: v1.inkPrimary · Subtitle color: v1.inkMuted · Subtitle style: italic.
  • Title-to-subtitle gap: 10px (Home + /shelves agree; /flagged drifts to 12 — pulling to 10).
  • Centering: flex column · alignItems center · textAlign center.
  • CTA slot: pass-through cta?: ReactNode, optional, rendered below subtitle (not used by any current callsite).

Out of scope

  • /my-shelf — uses AddFindTile (interactive card), not a text-block empty state. Not a primitive candidate.
  • Skeleton states (data === undefined) — separate primitive, not unified here.
  • Entrance animation — /shelves wraps its empty in motion.div; primitive stays static. Callers compose motion if they want it.

Net visual delta — small but real

  • Title size: /flagged drops 2px (24 → 22); Home + /shelves gain 2px (20 → 22). All converge to 22.
  • paddingTop on top-level surfaces: all three jump to 135 (= MASTHEAD_HEIGHT + 32). Home was 72 (sat too tight under masthead at the new 90px-grid system); /shelves was 80 (close); /flagged was 60 (tightest). Unified 135 reads more open without feeling lost — and removes the per-surface guesswork that paid for the drift.
  • /shelf/[slug] subtitle shrinks 1px (15 → 14) and gains a maxW (none → 260). Both small. Single-line Lora at 14 vs 15 is barely perceptible at this length.
  • Voice retention: all 4 copies preserved exactly. Per D3.

If the visual delta from D6 (paddingTop normalization) feels too far

  • Alternative: split the default into two — "top-of-page" (= MASTHEAD_HEIGHT + 32, used by Home + /shelves) vs "comfortable" (= MASTHEAD_HEIGHT only, no extra, used by /flagged where the action affordance lives in the masthead so the empty wants tighter clearance).
  • Or accept the converge and revisit if iPhone QA reads as "too low on /flagged."
  • My read: pick the converge. The current 60/72/80 split is unjustified drift. If one surface needs a custom value, pass clearance={60} explicitly; the prop already supports it.
