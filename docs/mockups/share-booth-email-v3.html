Window email — v3 info bar (post-it retired)

Pivot after v2.2 kept failing Gmail web + iOS Gmail QA. The rotated overlap post-it fights two Gmail filters at once (position:absolute stripping + SVG child filtering). v3 replaces it with a two-cell info bar below the banner — plain HTML table, zero position tricks, universal mail-client compat. Booth + mall read as one semantic unit: "this booth is at that mall."

Pivot Why the post-it has to go

  • Bug v2.0 (session 52 first pass): post-it as a styled div absolutely positioned inside a position: relative; overflow: hidden banner. Apple Mail + iOS Mail rendered correctly. Gmail web stripped position: absolute → post-it fell to flow position → clipped by overflow: hidden, invisible. iOS Gmail stripped position + probably overflow → post-it flowed below the banner image, not overlapping.
  • Bug v2.1 (session 52 second pass): refactored to v1 negative-margin overlay pattern (post-it sibling of banner, pulled up via margin-top: -108px, text-align: right + display: inline-block). David re-tested Gmail — still broken (same issue; the negative-margin trick also depends on inline-block/text-align flow that Gmail mangles in some variant).
  • Pivot Root truth: any post-it-over-banner overlap requires position or layout primitives Gmail is hostile to. Two rounds of clever workarounds, two rounds of failure on the client that matters most. The gesture is fighting the medium.

What v3 does differently

  • Fix No overlap, no rotation, no position tricks. Booth number moves into a horizontal info bar below the banner, paired with the mall name and address in a second cell. Pure HTML <table> — the oldest, most forgiving email primitive. Renders identically in Outlook 2007, Gmail web, iOS Gmail, Yahoo, Apple Mail, every other client anyone has used in the last 20 years.
  • Fix Semantic improvement. Booth number + mall info used to be split across two primitives (post-it overlay + separate pin-prefixed mall line below). Now they're one element: "Booth 123 / Americas Antique Mall, 5252 Bardstown Rd". "This booth is at that mall" reads as one thought. That was always the real relationship.
  • Fix Retires one other drift point quietly: the standalone mall-location line (pin glyph + mall name + address) is absorbed into the info bar. No more separate "location row" below the banner. One less primitive to maintain.

What stays locked from v2.2

  • SMALL masthead (13px uppercase Georgia 600 wordmark + 10px italic tagline)
  • Opener: "You've received a personal invite."
  • Eyebrow: "Step inside a curated booth from"
  • Vendor name hero: IM Fell 32px/400 (via Google Fonts link)
  • Window grid (3×2 tiles with sys-font captions)
  • Closer block + green pill CTA + footer
  • senderMode retirement from the email template (still server-side for rate limits)

What's gone

  • The post-it gesture in the email. Exists in the app (/shelf BoothHero) but retires from emails.
  • The teardrop PinGlyph SVG from the old standalone mall line. Info bar uses a text-only treatment; the mall is self-evidently a physical place from the name + dotted-underline address link.
  • Rotation, overlap, position: absolute, transform: rotate, SVG children — all gone from the banner/info-bar subtree.

Three variants below — pick one (or redirect)

  • A Attached info bar (sketch-faithful). Banner image + info bar share the same rounded outer frame with a hairline divider between them. Reads as one composed unit. Matches David's sketch most literally.
  • B Separate paper-wash card. Banner gets its own full border-radius. Info card floats below as a separate primitive with warm paper-wash fill (#fff9e8). Feels lighter, more breathable. Slight loss of "one unit" unity vs. A.
  • C Minimalist — no box chrome. Banner + two aligned columns below with only a thin vertical hairline between them. No outlined box at all. Quietest option. Booth number + mall sit directly on paper.

Questions for your call

  • Q1 Variant A / B / C? (Or a hybrid — e.g. "A's attachment but B's warm fill.")
  • Q2 Booth cell width — 32% feel right? Pulls the eye to the number. Alternative: 28% (tighter) or 38% (more balanced). Sketch shows roughly 30%.
  • Q3 Booth eyebrow copy — "Booth" or "BOOTH" (uppercase) or retire the label entirely and let the numeral stand alone? The sketch just has "Booth 123"; mockups below use italic "Booth" (title-case IM Fell).
