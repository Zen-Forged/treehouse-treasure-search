Frame comparison

Frame A — trailing in metadata block Glyph 14×14 inside the metadata strip, right-aligned bottom. Tap target: 18×18 pseudo-padded.
Pro: minimal layout disturbance — metadata stays centered with glyph as decorative trailing element.
Pro: reads as "this find" not as a separate affordance row.
Con: small tap target (18×18) — may need 32px hit-area expansion via padding to meet WCAG 44×44.
Con: compresses metadata vertical space slightly.
Frame B — dedicated 24×24 slot at frame bottom-right Glyph 16×16 inside a 24×24 slot below the metadata block, right-aligned. Metadata gets 26px right padding so it doesn't collide with the slot.
Pro: clear affordance — visually separates the save action from "this is a find" content.
Pro: larger tap target (24×24, expandable to ~36×36 with hit-area padding).
Con: +6px page height per row (3 rows = 18px; 5 rows = 30px) — denser feed becomes longer.
Con: metadata block right padding shifts text-center anchor 13px left; titles may read off-balance.
Frame C — REJECTED (on-photo top-right, baseline pattern from Home + /flagged) Shown for contrast. 28×28 frosted bubble overlaid on photo top-right (existing PolaroidTile.topRight slot).
Your rejection: "icons are in rows of 3 and adding to the image clutters it up." At /shelf/[slug] tile size (~115px wide), the bubble visually competes with photo content.
Note: this slot stays canonical for Home + /flagged where tiles are larger and a single-bubble overlay reads cleanly. /shelf/[slug] is the exception.
