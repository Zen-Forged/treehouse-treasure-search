Add Find — tag capture step

A new optional step between item-photo and review. Vendor photographs the inventory tag; Claude vision pulls the title + price; the review page prefills with badges showing what came from the tag versus what's still manual. Skip is a one-tap fallback to today's flow.

What this mockup is committing to

D1–D8 are already locked from chat. This mockup is the design surface for the four still-open subordinate decisions:

Already locked (architecture)

  • D1 Storage: tag photo discarded after extraction (no Supabase upload, no schema change)
  • D2 API: new /api/extract-tag mirrors /api/post-caption shape — Claude Sonnet 4.6 vision, mock fallback, rate limit, base64 input
  • D3 Flow: new page /post/tag between AddFindSheet and /post/preview
  • D4 Skip: lives on the tag page itself (primary photo CTA + secondary skip link). AddFindSheet stays unchanged
  • D5 Failure: mirror /api/post-caption — source: "claude" success / source: "mock" + reason: "error" on failure surfaces the existing <AmberNotice> on the preview page
  • D6 Title priority: tag title overrides Claude title; Claude caption still generated from item photo
  • D7 Price format: numeric or null; null → manual entry
  • D8 Caption timing: skip flow generates caption on preview mount (today's behavior); tag flow runs extract-tag on /post/tag and post-caption in parallel so preview lands with everything ready
Visual register: /post/tag reuses Mode C masthead chrome from /post/preview (38px back bubble, IM Fell 24 title, italic 14 subhead). The "from tag" badge introduces a new primitive — small inline pill next to the field label. Variant α below proposes green-on-green pill with Tag glyph + "from tag" italic; alternatives β (icon-only) and γ (ash-colored "auto" chip) shown below the frames.
Decisions to make:

  5. "From tag" badge — α (pill with Tag icon + "from tag" text), β (icon-only), or γ (ash "auto" chip)?
