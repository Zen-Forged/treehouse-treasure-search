Booth-numeral font system

Per session 75 redirect, item 7. The 1-vs-I confusion in IM Fell is biting at small numeral sizes — especially with mixed alphanumeric booth IDs like D19 or A1. Goal: pick a numeral font that disambiguates digits from letters, settle the rule, and apply it consistently across booth numerals + counts. IM Fell stays for headers + editorial voice.

Decisions to lock

Decision Recommendation
C1. The rule Yes — IM Fell for headers + editorial voice; numeral font for booth numerals + counts; sans for body. Headers = page titles, vendor names, mall names, eyebrows, ViewToggle, BoothCloser. Numerals = booth numbers (Variant B lockup, post-it 36px, find-detail post-it) + count chips (MallScopeHeader 22px, /flagged count).
C2. Font choice Times New Roman. Already in the codebase as FONT_POSTIT_NUMERAL (session v1.1l, post-it 36px exception, named in CLAUDE.md). Battle-tested at 36px. Disambiguated 1. No webfont load — ships with iOS, macOS, Windows. Playfair Display is a clean alternative but adds a Google Fonts request and shifts the visual register slightly more modern than the editorial baseline. Frames below let you compare directly.
C3. Scope of the swap Booth numerals AND counts. Counts (MallScopeHeader "47 finds across", /flagged count) share the same legibility risk at 22px italic. Pulling them onto the numeral font keeps the rule clean: "headers IM Fell; numbers numeral font." Alternative: numerals only, leave counts on IM Fell — splits the rule.
C4. Token name Rename FONT_POSTIT_NUMERAL → FONT_NUMERAL. Current name implies "post-it only," which is exactly the trap that left other numerals on IM Fell. Single rename + grep sweep.
C5. Italic posture for counts Current count is non-italic at 22px set inside an italic IM Fell eyebrow. Keep it non-italic — italic Times New Roman at 22px reads as old-fashioned. No change.
