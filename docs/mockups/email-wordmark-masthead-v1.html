Email wordmark masthead — v1

Reverses the session-52 v4 "no body masthead" decision. Hosted /wordmark.png as the brand anchor across all four templates: receipt, approval, window-share, and the Supabase OTP. Frame 1 chooses the size — Frame 2 verifies it across the four templates.

What this mockup is asking you to judge

  • DECIDE Wordmark size. S (160px wide, ~85px tall) reads quiet — brand-anchor without competing for the body. M (200px) is the natural midpoint. L (240px) is loudest — hard to miss, but eats vertical real-estate that used to belong to the body.
  • DECIDE Hairline divider below masthead. Mockup shows ON. Cleaner separation between brand chrome + body. OFF is also valid (more "letterhead" feel, less "section header" feel).

What's already locked

  • Image source: https://app.kentuckytreehouse.com/wordmark.png (hosted, served by Vercel — no CDN, but cached by mail clients after first load).
  • Centered, paperCream background, 32px top + 22px bottom padding.
  • alt="Treehouse Finds" so Gmail's image-block-by-default still shows the brand name as text.
  • Applies to all four templates via renderEmailShell() in lib/email.ts + the Supabase OTP HTML in docs/supabase-otp-email-templates.md.

What this reverses

  • Session 52 Q-011 v4 — "sender envelope already names the brand; re-identifying in the body added ~45px of vertical weight for zero new information." Reversed. The argument was right; the cost of failure is just low enough that David is willing to pay it for stronger brand presence.

Email-parity audit (per MASTER_PROMPT.md §Design Agent)

  • PRE-SHIP Image-rendering check across: Gmail web, Gmail iOS, Apple Mail iOS, Apple Mail macOS. Test send to David's inbox via real Resend codepath after wiring lands. Gmail blocks images by default — first-time recipients see the alt text "Treehouse Finds" until they whitelist the sender.
  • FLAG public/wordmark.png is 455KB unoptimized (operational backlog item from session 88). Email gets it inlined per send. Worth optimizing before this lands — pngquant --quality 70-90 wordmark.png typically cuts 60-80% for a wordmark with limited color palette.
