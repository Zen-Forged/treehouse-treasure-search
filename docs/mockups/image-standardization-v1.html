Image Standardization — E + B + A

Standardize photo height across uniform-grid surfaces. Three changes bundled: caption min-height, object-position: center top, and slot ratio flip from 4/5 portrait → 1/1 square.

Decisions — confirm or override

Locked (David approved): bundle E + B + A; mockup-first.

D1 — Slot ratio. Flip from aspect-ratio: 4/5 (portrait) to aspect-ratio: 1/1 (square) on FindTile / WindowTile / ShelfTile / ShelfCard. Squares crop nothing; portraits lose top/bottom equally. Booths VendorCard already uses 1/1 — this brings the find-tile family into the same idiom.

D2 — Crop position. Add object-position: center top on every <img> in the standardized tiles. Phone-portrait photos typically frame the subject in the upper half; cropping from the bottom preserves what matters.

D3 — Caption min-height. Caption block gets min-height: 56px so a 1-line title without price doesn't shrink the card relative to a 2-line title with price. Cards in any row now align bottom-to-bottom.

Q1 (open) — Should Mall page MallPostCard get the same flip? It currently uses paddingBottom: 75% (4:3 horizontal). Out of scope today since that page is on v0.2 dark theme deferred from session 71 standup, but worth noting if/when it migrates.

Q2 (open) — Discovery feed (masonry) is intentionally variable-height. Confirm we do NOT touch it — variable height is the point of the masonry rhythm.

BEFORE — current state (4/5 portrait slot, center crop)

All three surfaces enforce aspect-ratio: 4/5 + object-fit: cover. Photos are uniform per row. The visible problems: (1) square-source photos zoom in heavily because cover scales to fill the taller slot, (2) caption length variance makes cards bottom-out at different rows.

What changes in code

  • FindTile (/flagged) — photo div: aspect-ratio: 4/5 → 1/1; image gains object-position: center top; caption div gains min-height: 56px.
  • WindowTile (BoothPage) — same flip + same min-height. Also applies to AddFindTile + PlaceholderTile dashed cells so the 3-up grid stays uniform.
  • ShelfTile (BoothPage) — same flip + same min-height.
  • ShelfCard rail tile (/find/[id]) — same flip + same min-height (parallel pattern with BoothPage tiles).
  • Booths VendorCard (/shelves) — already 1/1; touches object-position only.
  • Mall page — out of scope (deferred from session 71).
  • Discovery feed — untouched (intentionally variable masonry rhythm).
  • Find Detail full-bleed photo — untouched (single-photo surface, no neighbor comparison).
